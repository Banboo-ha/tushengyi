# AI 产品海报生成 H5 产品需求文档 PRD

**版本**：V1.0  
**端**：手机端 H5  
**阶段**：第一阶段最小 MVP / 体验验证版本  
**核心能力**：用户注册登录 → 新用户赠送积分 → 上传产品图 → 上传其他参考图 → 填写文案 → 选择风格 → 消耗积分生成海报 → 查看结果 → 二次修改 → 保存作品 → 个人中心查看作品与积分

---

## 1. MVP 闭环判断

### 1.1 已确定 MVP 功能

| 模块 | 功能 |
|---|---|
| 用户系统 | 用户注册登录，采用用户名密码方式 |
| 积分系统 | 新用户赠送积分，生成和二次修改消耗积分 |
| 图片上传 | 上传产品图，最多 4 张 |
| 参考资料 | 上传其他参考图，如背景、Logo、模板、品牌风格，最多 4 张 |
| 文案填写 | 填写主标题、副标题、卖点文案 |
| 风格选择 | 选择 4 个基础风格 |
| AI 生成 | 调用 API 生成海报 |
| 结果查看 | 查看生成结果 |
| 二次修改 | 输入修改意见，再次生成 |
| 作品系统 | 保存作品，查看作品 |
| 个人中心 | 查看用户信息、积分、作品入口 |

### 1.2 判断结论

这套 MVP 可以形成完整的**产品体验闭环**，但暂时不是完整的**商业付费闭环**。

完整体验路径为：

```text
注册 / 登录
→ 获得新用户积分
→ 上传产品图
→ 上传其他参考图
→ 填写文案
→ 选择风格
→ 消耗积分生成海报
→ 查看生成结果
→ 二次修改
→ 保存作品
→ 在个人中心查看作品与积分
```

该闭环可以验证用户是否愿意使用、生成质量是否达标、二次修改是否有价值、作品保存是否有需求。

但由于第一阶段暂不包含支付、会员、积分购买，因此无法验证付费意愿、会员转化率和积分包价格策略。

### 1.3 第一阶段建议补充的轻量能力

为了让 MVP 更稳定，建议补充以下轻量能力：

| 功能 | 必要性 | 说明 |
|---|---|---|
| 积分不足提示 | 必须 | 有积分消耗就必然会出现积分不足，需要明确提示 |
| 积分明细 | 建议加入 | 用户需要知道积分来源和消耗原因 |
| 生成失败退还 | 必须 | AI 接口失败时需要自动退还积分，避免用户不信任 |
| 后台人工补积分 | 建议加入 | 便于灰度测试、内部测试和种子用户运营 |

## 2. 产品定位

### 2.1 产品一句话定位

一款面向小商家、运营人员、内容创作者的手机端 AI 产品宣传海报生成工具。用户只需要上传产品图、填写产品文案、选择风格，即可快速生成具有商业广告质感的产品海报，并支持通过自然语言进行二次修改。

### 2.2 产品核心价值

| 价值 | 说明 |
|---|---|
| 降低设计门槛 | 用户不需要会 PS，不需要懂复杂设计软件 |
| 降低 AI 使用门槛 | 用户不需要写复杂提示词，只需要填写结构化内容 |
| 提高出图效率 | 手机端即可完成产品海报生成 |
| 支持持续修改 | 不满意可以输入修改意见继续生成 |
| 适合商业场景 | 面向产品宣传、电商主图、小红书种草、本地生活推广 |

---

## 3. 产品目标

### 3.1 第一阶段目标

第一阶段不是商业化版本，而是**体验验证 MVP**。

核心目标：

1. 验证用户是否可以顺利完成一次海报生成；
2. 验证上传图 + 文案 + 风格选择的流程是否足够简单；
3. 验证生成结果是否具备保存和传播价值；
4. 验证二次修改是否能提升用户满意度；
5. 验证积分消耗机制是否能被用户理解。

### 3.2 长期目标

逐步发展为手机端 AI 商业视觉生成平台，覆盖：

- 产品宣传海报；
- 电商主图；
- 小红书封面；
- 门店活动图；
- 节日促销图；
- 品牌宣传图；
- 企业批量营销物料。

## 4. 目标用户

| 用户类型 | 需求 | 典型场景 |
|---|---|---|
| 小商家 / 个体商户 | 快速制作宣传图 | 朋友圈、社群、小红书、本地推广 |
| 电商运营 | 快速制作商品主图 | 淘宝、拼多多、抖店、视频号小店 |
| 本地生活商家 | 制作活动海报 | 餐饮、民宿、美容、鲜花、宠物店 |
| 自媒体 / 带货达人 | 制作种草图 | 小红书封面、短视频封面、产品推荐 |
| 企业市场人员 | 快速做基础营销物料 | 新品推广、活动预热、线索收集 |

### 4.1 用户特征

- 设计能力不强；
- 不熟悉 AI 生图提示词；
- 希望用手机快速完成；
- 更关注结果，而不是复杂编辑过程；
- 对商业质感、清晰排版、产品突出有要求；
- 希望不满意时可以继续修改。

| 功能 | 后置原因 |

| 企业版 | 需要更完整的账号、权限、模板体系 |
| 复杂编辑器 | MVP 先用自然语言二次修改替代 |

## 8. 核心用户流程

### 8.1 新用户首次生成流程

```text
用户访问 H5
→ 进入首页
→ 点击立即生成
→ 跳转登录 / 注册
→ 用户名密码注册
→ 注册成功，系统赠送积分
→ 进入上传产品图页
→ 上传 1-4 张产品图
→ 上传其他参考图，可跳过
→ 填写主标题 / 副标题 / 卖点
→ 选择基础风格
→ 确认消耗积分
→ 创建生成任务
→ 等待生成
→ 查看生成结果
→ 保存作品
```

### 8.2 老用户生成流程

```text
用户打开 H5
→ 自动识别登录状态
→ 首页展示当前积分
→ 点击立即生成
→ 上传产品图
→ 上传参考图或跳过
→ 填写文案
→ 选择风格
→ 确认生成
→ 查看结果
→ 保存或继续修改
```

### 8.3 二次修改流程

```text
用户在生成结果页点击继续修改
→ 进入二次修改页
→ 输入修改意见
→ 系统检测积分是否足够
→ 积分足够则创建修改任务
→ 生成新版本
→ 展示新结果
→ 用户保存作品
```

### 8.4 积分不足流程

```text
用户点击生成或二次修改
→ 系统检测积分余额
→ 积分不足
→ 弹出积分不足提示
→ 用户返回首页 / 查看积分明细 / 联系客服
```

### 8.5 生成失败流程

```text
用户提交生成任务
→ AI 接口生成失败
→ 系统标记任务失败
→ 自动退还积分
→ 展示失败原因
→ 用户可重新生成或更换图片
```

## 9. 页面需求

### 9.1 登录 / 注册页

**页面目标**：让用户通过用户名和密码完成注册或登录。

**页面元素**：

- Logo；
- 用户名输入框；
- 密码输入框；
- 确认密码输入框，注册模式显示；
- 登录按钮；
- 注册按钮；
- 登录 / 注册模式切换入口；
- 用户协议勾选；
- 错误提示。

**规则**：

| 场景 | 提示 |
|---|---|
| 用户名为空 | 请输入用户名 |
| 密码为空 | 请输入密码 |
| 密码长度不足 | 密码至少 6 位 |
| 注册时两次密码不一致 | 两次输入的密码不一致 |
| 用户名已存在 | 用户名已被注册 |
| 登录失败 | 用户名或密码错误 |

### 9.2 首页

**页面目标**：让用户理解产品能力，并快速开始生成。

**页面元素**：

- 产品名称；
- 当前积分；
- 立即生成按钮；
- 产品能力说明；
- 最近作品入口；
- 4 个基础风格展示；
- 底部导航。

**点击逻辑**：

| 点击对象 | 结果 |
|---|---|
| 立即生成 | 已登录进入上传产品图页，未登录进入登录页 |
| 我的作品 | 进入作品库 |
| 积分 | 进入积分中心 |
| 我的 | 进入个人中心 |

### 9.3 上传产品图页

**页面目标**：让用户上传用于生成海报的核心产品图。

**页面元素**：

- 顶部进度：1/4 上传产品图；
- 页面标题：上传你的产品图；
- 图片上传区域；
- 已上传图片列表；
- 上传数量提示；
- 继续按钮。

**规则**：

| 规则 | 说明 |
|---|---|
| 最少上传 | 1 张 |
| 最多上传 | 4 张 |
| 支持格式 | jpg、jpeg、png、webp |
| 单张大小 | 建议不超过 10MB |
| 数量达到 4 张 | 隐藏或禁用继续上传按钮 |
| 未上传点击下一步 | 提示“请至少上传 1 张产品图” |
| 上传失败 | 提示“上传失败，请重试” |

### 9.4 其他参考图上传页

**页面目标**：让用户补充背景、Logo、品牌风格、模板样式等辅助参考图。

**页面元素**：

- 顶部进度：2/4 上传参考图；
- 页面标题：上传其他参考图；
- 页面说明：可上传背景、Logo、品牌风格、模板样式等；
- 图片上传区域；
- 已上传参考图列表；
- 参考图类型标签；
- 跳过按钮；
- 下一步按钮。

**参考图类型**：

- 背景参考；
- Logo 参考；
- 品牌风格；
- 排版模板；
- 色彩参考；
- 其他。

**规则**：

| 规则 | 说明 |
|---|---|
| 是否必填 | 否 |
| 最多上传 | 4 张 |
| 可跳过 | 是 |
| 支持格式 | jpg、jpeg、png、webp |
| 上传后可删除 | 是 |
| 上传后可选择类型 | 是 |

### 9.5 填写文案页

**页面目标**：让用户填写生成海报所需的基础文案。

**字段规则**：

| 字段 | 是否必填 | 字数建议 |
|---|---|---|
| 主标题 | 必填 | 2-20 字 |
| 副标题 | 选填 | 2-30 字 |
| 卖点文案 | 选填 | 10-100 字 |

**空值规则**：

| 情况 | 处理 |
|---|---|
| 主标题为空 | 不允许下一步 |
| 副标题为空 | 允许 |
| 卖点为空 | 允许 |
| 文案过长 | 提示精简 |

### 9.6 选择风格页

**页面目标**：让用户选择海报生成风格。

**第一阶段 4 个基础风格**：

| 风格 | 适合场景 |
|---|---|
| 高级商业广告 | 产品广告、品牌宣传、食品饮品、科技产品 |
| 小红书种草风 | 民宿、美妆、生活方式、探店 |
| 电商主图风 | 商品销售、平台主图、促销图 |
| 极简高级风 | 家居、香氛、服装、设计类产品 |

**第一阶段画幅比例建议**：

- 1:1；
- 3:4；
- 4:5。

后续可扩展：

- 9:16；
- 16:9；
- 21:9。

### 9.7 确认生成页

**页面目标**：让用户在提交前确认信息和积分消耗。

**页面元素**：

- 产品图缩略图；
- 参考图缩略图；

### 9.8 生成等待页

**状态文案**：

**状态规则**：

|---|---|
| pending | 准备生成 |
| running | 生成中 |
| success | 跳转结果页 |
| failed | 跳转失败页 |
| timeout | 提示服务繁忙 |

### 9.9 生成结果页

**页面目标**：展示 AI 生成结果，并引导保存或二次修改。

**页面元素**：

- 生成完成提示；

- 版本号；
- 保存作品按钮；
- 继续修改按钮；
- 重新生成按钮；
- 返回首页按钮。

**操作规则**：

| 操作 | 结果 |
|---|---|
| 保存作品 | 保存到作品库 |
| 继续修改 | 进入二次修改页 |
| 重新生成 | 再次调用生成任务，消耗积分 |
| 返回首页 | 回到首页 |

### 9.10 二次修改页

**快捷修改标签**：

- 背景更高级；
- 标题更大；
- 去掉文字；
- 换成小红书风；
- 换成电商主图；
- 画面更简洁；
- 色彩更明亮。

**规则**：

| 条件 | 结果 |
|---|---|
| 修改意见为空 | 不允许提交 |
| 积分不足 | 弹出积分不足提示 |
| 生成成功 | 生成新版本 |
| 生成失败 | 退还积分 |

### 9.11 作品库页

## 22. MVP 验收标准

### 22.1 用户注册登录

- 用户可以通过用户名密码注册；
- 注册成功后自动登录；
- 新用户自动获得积分；
- 用户可以退出登录；
- 用户可以再次登录。

### 22.2 上传产品图

- 用户至少上传 1 张产品图；
- 最多上传 4 张产品图；
- 上传后可以预览；
- 上传后可以删除；
- 超过 4 张不允许继续上传。

### 22.3 上传参考图

- 用户可以上传 0-4 张参考图；
- 用户可以跳过；
- 用户可以给参考图选择类型；
- 上传后可以预览和删除。

### 22.4 文案填写

- 主标题必填；
- 副标题选填；
- 卖点选填；
- 文案过长有提示。

### 22.5 风格选择

- 用户必须选择 1 个风格；
- 第一阶段提供 4 个基础风格；
- 用户可以查看当前积分和预计消耗。

### 22.6 生成海报

- 积分足够时可以生成；
- 积分不足时不可生成；
- 生成任务可以异步查询；
- 生成成功展示图片；
- 生成失败退还积分。

### 22.7 二次修改

- 用户可以基于当前作品输入修改意见；
- 二次修改消耗积分；
- 成功后生成新版本；
- 失败后退还积分。

### 22.8 作品保存

- 用户可以保存生成作品；
- 作品库可以查看已保存作品；
- 作品详情可以查看版本。

## 23. 最终建议

第一阶段 MVP 是成立的，但应明确它的定位：

```text
V1.0 是体验验证 MVP，不是商业化完整版本。
```

第一阶段最重要的不是把支付、会员、模板商城全部做完，而是先验证：

1. 用户能否低门槛完成一次生成；
2. 生成结果是否达到宣传图基本要求；
3. 二次修改是否能提升结果满意度；
4. 积分机制是否能让用户理解成本；
5. 作品保存是否能形成复用价值。

推荐最终 MVP 闭环为：

```text
注册登录
→ 新用户得积分
→ 上传产品图
→ 上传参考图
→ 填写文案
→ 选择风格
→ 消耗积分生成
→ 查看结果
→ 二次修改
→ 保存作品
→ 个人中心查看积分和作品
```

建议第一阶段务必补齐：

- 积分不足提示；
- 积分明细；
- 失败退还；
- 后台人工补积分。

这样产品就可以真实上线测试，并为后续支付、会员、模板中心和企业版打基础。
